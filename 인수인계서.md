# 마인드팔레트 (MindPalette) 인수인계서

## 1. 프로젝트 개요
* **프로젝트명:** MindPalette (마인드팔레트)
* **주제:** AI 기반 비언어적 정서 외부화 플랫폼 (AI-based Non-verbal Emotional Externalization Platform)
* **목적:** 사용자가 거칠게 그린 스케치와 감정 키워드를 바탕으로 AI가 예술 작품으로 변환해주는 디지털 미술 치료 솔루션
* **저장소:** [GitHub Repository](https://github.com/jiha-hub/Jiha_DigitalAI_ArtTherapy)

## 2. 주요 기술 스택
* **Language:** Python 3.10+
* **Framework & UI:** Gradio 4.0+
* **AI & Deep Learning:** PyTorch, Hugging Face `diffusers`, `transformers`
* **Core Models:**
  * SDXL Base 1.0 + T2I-Adapter (스케치 기반 뼈대 생성)
  * Kandinsky 2.2 Inpainting (부분 수정)
  * 치료적 화풍을 위한 맞춤형 LoRA
* **배포 환경:** Hugging Face Spaces (GPU 환경 필요)

## 4. 로컬 환경 실행 가이드

1. **가상환경 생성 및 활성화** (권장)
   ```bash
   python -m venv .venv
   .venv\Scripts\activate  # Windows 기준
   ```
2. **패키지 설치**
   ```bash
   pip install -r requirements.txt
   ```
3. **애플리케이션 실행**
   ```bash
   python src/app.py
   ```
   실행 후 터미널에 나타나는 로컬 주소(예: `http://127.0.0.1:7860`)로 접속합니다.

## 5. 배포 가이드 (Hugging Face Spaces)
1. Hugging Face에 접속하여 새로운 `Gradio` Space를 생성합니다.
2. 해당 Space를 이 GitHub Repository(`https://github.com/jiha-hub/Jiha_DigitalAI_ArtTherapy`)와 연동합니다.
3. `README.md` 상단에 설정된 메타데이터(`app_file: src/app.py`)를 기반으로 자동 빌드 및 배포가 진행됩니다.
4. 필요시, Space의 하드웨어 설정을 GPU 환경으로 변경해야 원활하게 모델 생성이 가능합니다.

## 6. 기타 참고사항
* 로컬에서 실행할 경우 `src/app.py` 16번째 줄의 `low_vram=True` 설정이 GPU 메모리 부족 현상을 방지합니다.
* 갤러리 내역은 `outputs/gallery_log.json` 파일에 저장되며, 업로드된 이미지 및 결과물 파일들은 해당 폴더에 관리됩니다.
* 기획 및 발표 자료는 동봉된 `미술치료ai_v4.pptx`와 `전체구성도.docx`를 참고하시기 바랍니다.
