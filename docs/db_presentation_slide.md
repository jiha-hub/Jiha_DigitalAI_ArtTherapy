# 🗄️ MindPalette: DB Design for Presentation (Single Slide View)

이 자료는 발표 슬라이드 한 장에 들어갈 수 있도록 시각화된 DB 설계 구조입니다.

| 테이블 분류 | 핵심 필드 (Attributes) | 연관 관계 (Relationships) |
| :--- | :--- | :--- |
| **USER** (환자) | `ID`, `닉네임`, `연령대`, `가입일` | 모든 테이블의 부모 (Master) |
| **BIOMETRICS** (생체) | `HR(심박수)`, `HRV(변이도)`, `측정일시` | **Artwork** 생성의 정서적 근거 |
| **ARTWORK** (작품) | `스케치경로`, `결과물경로`, `프롬프트`, `스케치비중` | **Diary**와 1:1 매칭 |
| **DIARY** (일기) | `내용`, `무드태그`, `위험단계(High-Risk Flag)` | 정성적 분석의 핵심 데이터 |

### 🛠️ 데이터 통합 아키텍처 (ERD 시각화 전략)
1. **동기화 (Sync)**: 환자가 그림을 그리는 동안 **BIOMETRICS**가 초 단위로 수집됩니다.
2. **결합 (Bond)**: 생성된 **ARTWORK**는 당시의 평균 심박수 지수와 자동으로 결합됩니다.
3. **통찰 (Insight)**: 상담사는 **DIARY**의 위험 단어 필터를 통해 환자의 응급 상태를 즉시 파악합니다.

> **Privacy Warning**: 모든 이미지와 생체 원천 데이터는 로컬 DB에 암호화 저장되며, 외부로 나가는 통계 리포트에는 `UUID` 처리된 익명 데이터만 사용됩니다.
