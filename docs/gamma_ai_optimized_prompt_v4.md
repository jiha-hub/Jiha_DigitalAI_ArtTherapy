### [본문 소스 텍스트]

#### I. 전체 구성도 (System Overview)
- 마인드팔레트는 사용자의 심리적 상태를 시각화하고 치유하는 4단계 프로세스를 가집니다.
- **프로세스 표**:
  1. **표현 (Expression)**: PidiNet + SDXL Adapter + Therapeutic LoRA (드로잉 변환)
  2. **분석 (Analysis)**: Bio-feedback Agent (Nurse + Data Agent) (정서 분석)
  3. **다듬기 (Refinement)**: Kandinsky 2.2 Inpainting (상징물 삽입 및 수정)
  4. **공유 (Consolidation)**: Social Gallery System (기록 및 지지적 댓글)

#### II. 논문 기반 (Academic Background)
**1. 배경**
- **문제제기 / 필요성**: 현대인의 스트레스로 인한 '정서적 실어증' 해결 필요. 언어 중심 상담의 한계를 넘는 비언어적 예술 치료의 접근성 확대 필요.

**2. 방법**
- **적용 기술**: 사용자의 원본 구도를 90% 이상 유지하는 **T2I-Adapter**와 심리적 안정을 위한 **LoRA(Low-Rank Adaptation)** 가중치 기술.
- **연구 목적 / 설정**: 사용자의 '자기 주체성(Self-Agency)'을 보존하면서 최적의 치유적 효능감을 얻을 수 있는 기술적 조건 확립.
- **연구방법**: 스케치 준수율(Conditioning Scale) 0.8~1.0 구간 분석 및 Kandinsky 2.2 기반 화풍 상속 루프 테스트.
- **논문 기반 설계 (Architecture)**: 엣지 검출기(PidiNet)와 예술적 생성기(SDXL)가 결합된 제어형 생성 파이프라인.

**3. 결과**
- **결과 해석**: 사용자의 선을 존중할 때 소유감(Sense of Ownership) 94% 이상 달성. 치료적 LoRA 적용 시 고채도 자극 감소 및 심리적 안정 효과 정량화.

#### III. 솔루션 기반 (Solution Integration)
**1. 솔루션 개요**
- **솔루션 컨셉**: "당신의 낙서가 위로의 마스터피스가 되는 곳, 마인드팔레트."
- **주요 기능**: AI 드로잉 변환, 생체 데이터 기반 감정 분석, 10대 치유 테마 인페인팅 가이드, 소셜 전시 갤러리.

**2. 솔루션 설계 및 구현 설명**
- **솔루션 아키텍처 (논문 + 솔루션)**: 
  - Input: 실시간 스케치 + 심박수 + 감정 데이터.
  - Core: SDXL(Base) + T2I-Adapter + Kandinsky(Inpaint) + PEFT(LoRA 로딩).
  - Storage: 로컬 암호화 저장 기반의 프라이버시 보호 구조.
- **주요 기능 및 흐름 설명**: 스케치 입력 후 LoRA 화풍 적용 -> 분석 에이전트의 치유 가이드 제안 -> 마스킹 영역에 긍정적 개체 삽입 -> 최종 갤러리 저장 및 SNS 초대장 공유.

**3. 솔루션 시연**
- **솔루션 화면 레이아웃**: 
  - 왼쪽: 내비게이션 사이드바 (표현/다듬기/전시관).
  - 중앙: 페인팅 에디터 및 AI 생성 결과 병렬 뷰.
  - 갤러리: 고전 액자 스타일의 큐레이션 카드 및 모바일 초대장 상세 보기.

**4. 코드 리뷰**
- **개발 환경**: Python 3.10 (Conda), CUDA 11.8+, Gradio 4.0+.
- **핵심 기술 구현**: `apply_lora()`를 통한 실시간 모델 가중치 융합 및 `handle_mod_with_style()` 프롬프트 상속 기술.
- **데이터 관리**: `gallery_log.json` 기반의 영속성 관리 및 데이터 익명화 처리.
